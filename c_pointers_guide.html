A pointer is a variable that stores the memory address of another variable. Think of memory as a row of numbered boxes — a pointer stores the box number. int *p declares a pointer to an int. &n gets the address of n. *p reads the value at that address.
pointer stores an address &n → address of n *p → value AT the address int *p → pointer to int

& (address-of): gives the memory address of a variable. Used when creating a pointer.

* (dereference): follows the pointer to access the value at that address. Can read OR write through it.

Remember: int *p in a declaration means "p is a pointer to int". *p in an expression means "value at p".
& → get the address * → follow the address *ptr = 99 → modify original * in declaration ≠ * in expression
pointer_arithmetic.c
int arr[] = {10, 20, 30, 40, 50};
int *p = arr;     // p points to arr[0]

printf("%d\n", *p);     // 10 — arr[0]
p++;               // move to next int (+4 bytes)
printf("%d\n", *p);     // 20 — arr[1]
p += 2;            // jump ahead 2 ints
printf("%d\n", *p);     // 40 — arr[3]
p--;               // step back one int
printf("%d\n", *p);     // 30 — arr[2]
array in memory — p walks through it

When you do p++ on an int*, p jumps 4 bytes (size of int) — not 1 byte. C scales pointer arithmetic by the size of the pointed type automatically. This is why pointers and arrays are so closely linked: arr[i] is exactly the same as *(arr + i).
p++ moves by sizeof(int)=4 arr[i] == *(arr+i) scaled by element size don't go out of bounds!
pointer_pitfalls.c
// ✅ NULL pointer — safe "no address" value
int *p = NULL;     // p points nowhere
if (p != NULL) *p = 5;  // safe check!

// ❌ BUG 1: Uninitialized pointer (wild pointer)
int *q;            // q has garbage address!
*q = 10;           // CRASH — undefined behavior

// ❌ BUG 2: Dangling pointer
int *r = (int*)malloc(sizeof(int));
free(r);           // memory freed
*r = 5;            // CRASH — dangling pointer!
r = NULL;          // ✅ fix: set to NULL after free

// ❌ BUG 3: Wrong type pointer
float f = 3.14;
int *ip = (int*)&f;  // misinterprets bytes!

Pointer bugs are among the hardest to debug in C. A bad pointer crash may happen far from where the bug was introduced. Always initialise pointers to NULL, check before dereferencing, and set to NULL after free().
init to NULL if no address yet wild pointer → crash dangling pointer → crash always check != NULL before *p

pointer size
// Pointer size depends on architecture, NOT the type it points to
sizeof(int *)   == 8   // 64-bit system
sizeof(char *)  == 8   // same!
sizeof(double*) == 8   // same!

// arr[i] and pointer arithmetic are identical:
arr[3]  ==  *(arr + 3)  // always true
&arr[3] ==  (arr + 3)  // address equivalence
