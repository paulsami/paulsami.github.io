int recurse(int n) {
  /* 1. Base case — stops the recursion */
  if (n == 0) return 1;
  /* 2. Recursive case — calls itself */
  return n * recurse(n - 1);
}

how the call stack works
Each call creates a new stack frame with its own local variables. Frames pile up as calls go deeper. When the base case is hit, frames pop off one by one — each returns its value to the caller above it.
recurse(3) calls recurse(2)
recurse(2) calls recurse(1)
recurse(1) calls recurse(0)
recurse(0) → returns 1 ← BASE
recurse(1) → returns 1×1 = 1
recurse(2) → returns 2×1 = 2
recurse(3) → returns 3×2 = 6
Recursion solves a problem by breaking it into a smaller version of the same problem. The key insight: if you know how to solve the problem for n-1, you can solve it for n. Without a base case, the function calls itself forever → stack overflow crash.
base case = stop condition recursive case = smaller subproblem no base case = stack overflow each call gets its own stack frame
factorial.c
int factorial(int n) {
  if (n == 0 || n == 1)
    return 1;  // base case
  return n * factorial(n-1); // recurse
}

fibonacci.c
int fib(int n) {
  if (n <= 1)
    return n;  // base: fib(0)=0, fib(1)=1
  return fib(n-1) + fib(n-2); // two calls!
}

Fibonacci makes two recursive calls per step — this creates a tree of calls that grows exponentially. fib(5) makes 15 calls, fib(10) makes 177 calls, fib(30) makes over 2 million. This is why the naive recursive version is O(2ⁿ). The fix is memoisation — cache results to avoid recomputing.
fib(n) = fib(n-1) + fib(n-2) O(2ⁿ) — exponential calls fib(2) computed repeatedly fix: memoisation or DP

int A(int m, int n) {
  if      (m == 0)
    return n + 1;        // base case 1
  else if (n == 0)
    return A(m-1, 1);   // base case 2
  else
    return A(m-1, A(m, n-1)); // nested!
}

The Ackermann function is famous for growing faster than any primitive recursive function. Even tiny inputs like A(4,2) produce astronomically large numbers. It is a classic example in theoretical computer science showing that not all computable functions can be expressed with simple loops — it requires deep recursion.
doubly recursive — nested A(m, A(m,n-1)) A(4,2) is incomputably large 3 cases, 2 base cases A(2,2) = 7 — try it!
quicksort.c
void quicksort(int a[], int lo, int hi) {
  if (lo >= hi) return; // base case
  int p = partition(a, lo, hi); // pivot index
  quicksort(a, lo, p-1);  // recurse left
  quicksort(a, p+1, hi);  // recurse right
}

Quick Sort uses divide and conquer recursion: pick a pivot, partition elements smaller/larger, then recursively sort both halves. Base case: subarray of length ≤ 1 is already sorted. Average O(n log n) — the recursion tree has log n levels and each level does O(n) work.
divide and conquer avg O(n log n) worst O(n²) — sorted input base: lo ≥ hi (size 0 or 1)
