A Practical Guide to main() Parameters
char *argv[]  vs  char **argv — two ways to write the same thing in C
Form A — Array notation classic
int main(int argc, char *argv[]) { // argv is an array of // char pointers }
Form B — Pointer notation equivalent
int main(int argc, char **argv) { // argv is a pointer to // a char pointer }
Form A — char *argv[] explained
1
argv
Name of the parameter. Holds the command-line arguments.
2
[] — array of
Declares argv as an array. The size is unspecified — determined at runtime by argc.
3
char *
Each element of the array is a pointer to a char (i.e., a C string). So argv is an array of strings.
✦
Reading style
Right-to-left: "argv is an array [] of pointers * to char". Most textbooks use this form.
Form B — char **argv explained
1
argv
Same name — holds the same command-line arguments as Form A.
2
** — pointer to pointer
argv is a pointer to a char*. The first * makes it a pointer; the second * means it points to another pointer.
3
char
The ultimate target type is char. So: argv → pointer → char* → char string.
✦
Reading style
Right-to-left: "argv is a pointer * to a pointer * to char". Common in systems/OS programming.

⚖️ Verdict — Which should you use?
🟣
char *argv[] — use this in exams and teaching. It is self-documenting: you can immediately see "array of string pointers". Preferred by most C textbooks and university syllabi.
🟢
char **argv — use this when you need to do pointer arithmetic like argv++ inside the function, or when emphasising the pointer-to-pointer relationship. Common in Unix/Linux systems code.
✅
Both are 100% correct and compile to identical machine code. The C standard explicitly permits both. The choice is purely a matter of style and intent.
identical at runtime char *argv[] → clearer for beginners char **argv → enables argv++ increment argv[argc] == NULL always

while loop

The while loop checks its condition before each iteration, making it perfect for situations where the number of repetitions depends on runtime conditions.

do-while loop

The do-while loop runs its body first and checks the condition afterward, guaranteeing at least one execution before the loop may terminate.

comparison

All three loops can express the same logic, but each is best suited to different scenarios: for for counted loops, while for condition-driven loops, and do-while when a loop must run at least once.
