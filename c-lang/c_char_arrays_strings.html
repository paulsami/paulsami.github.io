what_is_string.c
// A string in C = char array ending with '\0'
char name[] = "HELLO";

// Same thing written manually:
char name2[] = {'H','E','L','L','O','\0'};

// '\0' is ASCII 0 — the null terminator
// Without it, printf keeps reading garbage!
printf("%s\n", name); // prints: HELLO
memory layout of "HELLO" — 6 bytes (5 chars + null terminator)

The null terminator '\0' (ASCII value 0) is what makes a char array a string. Functions like printf, strlen, strcpy scan characters until they hit '\0' — that's how they know where the string ends.
string = char array + '\0' "HELLO" takes 6 bytes no '\0' = undefined behavior '\0' is ASCII 0, not '0'(48)
string_declare.c
// 1. From string literal (auto '\0')
char s1[] = "Hi";        // size = 3

// 2. Fixed size — must leave room for '\0'
char s2[10] = "Hi";      // [H,i,\0,0,0...]

// 3. Pointer to string literal (read-only!)
char *s3 = "Hi";        // cannot modify!

// 4. Empty buffer for user input
char buf[50];           // reserve space
fgets(buf, 50, stdin); // safe input

// strlen = chars NOT counting '\0'
strlen("Hi") == 2;      // true
memory: char s2[10] = "Hi"
Key differences:

char s[]="Hi" — array, modifiable, size auto-computed (3)

char s[10]="Hi" — array, modifiable, extra slots filled with \0

char *s="Hi" — pointer to read-only literal in memory. Trying to modify crashes!
always size+1 for '\0' char* literal is read-only
string.h functions
#include <string.h>

char a[20] = "Hello";
char b[20] = "World";

strlen(a)           // → 5   (length)
strcpy(a, b)        // a = "World"
strcat(a, " !")     // a = "World !"
strcmp(a, b)        // 0=equal, neg/pos
strchr(a, 'o')     // pointer to 'o'
strstr(a, "orl")   // pointer to "orl"
strncpy(a,b,3)     // copy only 3 chars (safer)
strncat(a,b,3)     // append only 3 chars

string_loop.c
char s[] = "hello";
int len = strlen(s);  // = 5

// Loop until '\0'
for (int i = 0; s[i] != '\0'; i++) {
    s[i] = s[i] - 32; // lowercase→uppercase
}                     // 'a'=97, 'A'=65, diff=32

printf("%s\n", s); // prints: HELLO

string_pitfalls.c
// ❌ BUG 1: No room for '\0'
char s[5] = "Hello"; // needs 6! overflow!

// ❌ BUG 2: strcmp with ==
char a[] = "hi", b[] = "hi";
if (a == b) // WRONG! compares addresses
if (strcmp(a,b)==0) // ✅ correct!

// ❌ BUG 3: Modifying a string literal
char *p = "hi";
p[0] = 'H'; // CRASH — read-only memory!

// ❌ BUG 4: scanf overflow
char buf[10];
scanf("%s", buf);       // dangerous
scanf("%9s", buf);      // ✅ limit to 9

Strings in C require careful memory management. Buffer overflows are one of the most common security vulnerabilities in C programs. Always allocate length + 1 bytes, and prefer fgets over scanf("%s") for user input.
always size = len + 1 use strcmp not == char* literals are read-only fgets for safe input
