real-world analogy
🏠 House analogy
A variable is like a house (it stores something).
A pointer is like a piece of paper with the house's address written on it.
You don't carry the house — you carry the address, and go there when needed.
📦 Memory analogy
RAM = a long row of numbered boxes (addresses).
Every variable occupies one or more boxes.
A pointer stores the box number (address) of another variable.
Why pointers exist
✦ Pass large data without copying
✦ Modify a caller's variable from inside a function
✦ Build dynamic structures (linked lists, trees)
✦ Arrays and strings are pointers under the hood

The key insight: a pointer is just an integer — the memory address of another variable. The type (int *, char *) tells C how many bytes to read when you dereference it, not the size of the pointer itself (always 8 bytes on 64-bit systems).
pointer = stores an address &x → address of x *p → value at that address pointer size = 8 bytes (64-bit)
defining_pointers.c
// Syntax: type *pointer_name;
int    *p;   // pointer to int
char   *c;   // pointer to char
float  *f;   // pointer to float
double *d;   // pointer to double

// Initialise: point to an existing variable
int n = 10;
int *ptr = &n;   // ptr = address of n

// Read through pointer (dereference)
int val = *ptr;  // val = 10

// Write through pointer
*ptr = 99;       // n is now 99

// Pointer to pointer
int **pp = &ptr; // pp → ptr → n
printf("%d\n", **pp); // prints 99

✅ correct pointer usage
int x = 5;
int *p = &x;  // init!
*p = 10;      // safe ✓
❌ dangerous pointer usage
int *p;      // uninitialised!
*p = 10;    // CRASH ✗
// p has garbage address
The * in a declaration (int *p) means "p is a pointer to int". The * in an expression (*p) means "dereference — give me the value at address p". These are two different uses of the same symbol. Always initialise a pointer before using it.
int *p — declaration *p — dereference in expression int **pp — pointer to pointer never use uninitialised pointer
self_ref_struct.c
// A struct that contains a pointer
// to its OWN type — self-referential
struct Node {
  int  data;          // the value
  struct Node *next;  // pointer to next Node
};

// Create nodes manually
struct Node a, b, c;
a.data = 10;  a.next = &b;
b.data = 20;  b.next = &c;
c.data = 30;  c.next = NULL; // end!

// Access fields
printf("%d\n", a.data);       // 10
printf("%d\n", a.next->data); // 20

A self-referential structure contains a pointer to its own type. This is the foundation of all linked data structures. Notice: struct Node *next is a pointer to a Node — not a Node itself (which would be circular and impossible in C). The pointer stores an address, which is just an integer.
struct contains pointer to itself node.data — access data field node.next — access pointer field NULL marks the last node

struct Node {

// Traverse: start at head, follow next until NULL

  printf("%d ", curr->data);

array vs linked list — key differences
aspect array linked list
memory contiguous (all together) scattered anywhere in memory
access O(1) — direct by index O(n) — must traverse from head
size fixed at compile time grows/shrinks dynamically
insertion O(n) — shift elements O(1) — just relink pointers
extra memory none each node stores a pointer too
linked by index arithmetic pointers (next field)
A linked list is a chain of nodes where each node contains data and a pointer to the next node. You only need to know the head (first node) — from there you can reach every other node by following next pointers. The last node's next is NULL — this is how you know you've reached the end.
head → node1 → node2 → NULL curr = curr->next to traverse -> operator: ptr->field NULL = end of list
